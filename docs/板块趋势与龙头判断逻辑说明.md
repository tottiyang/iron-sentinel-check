# 铁血哨兵 V3 — 板块趋势与龙头判断逻辑说明

> 本文档详细说明 `[9] 板块趋势向上` 和 `[10] 龙头活跃` 两项审核的完整内部逻辑，包括数据来源、评分规则、阈值判定、输出格式及已知问题修复记录。

---

## 一、[9] 板块趋势向上（V3）

### 1.1 整体流程

```
个股代码
  → 查询多维度板块归属（申万L1/L2/L3 + EM概念 + EM行业）
  → 双轨并行获取板块数据（NeoData指数 + 成分股实时计算）
  → 筛选核心板块（综合得分排序，确保多样性）
  → 对最佳核心板块进行 7 维度评分
  → 输出 CheckResult
```

### 1.2 板块归属（Step 1）

| 来源类型 | 层级 | 权重 |
|---------|------|------|
| 申万L1 | 行业一级 | 0.90 |
| 申万L2 | 行业二级 | 0.85 |
| 申万L3 | 行业三级 | 0.80 |
| EM概念 | 概念板块 | 0.70 |
| EM行业 | 行业板块 | 0.65 |

从 `stock_industry` / `concept_boards` / `industry_boards` 等表查询，按权重降序排列。

### 1.3 双轨数据获取（Step 2）

**轨道 A（NeoData）**：通过自然语言查询板块指数数据
- 今日涨幅、5日涨幅、20日涨幅、换手率、量比、成交额
- 缓存 180 秒

**轨道 B（成分股计算）**：
- 从 DB 取该板块成分股列表（申万通过 `stock_industry` 表直接查；EM/SINA 通过 `stock_concept` / `stock_industry_board` 查）
- 限制最多取 **15 只**成分股（避免请求过多导致超时）
- 并发请求（8 线程）每只成分股的实时数据：
  - 今日涨幅：腾讯实时行情
  - 5日/20日涨幅：新浪日K线
  - 量比：今日成交量 / 近5日平均成交量
- 计算板块平均：
  - `chg_pct` = 成分股平均今日涨幅
  - `up_ratio` = 上涨家数占比
  - `limit_up_count` = 涨停家数（≥9.9%）
  - `avg_gain_5d` / `avg_gain_20d` = 平均5日/20日涨幅
  - `avg_volume_ratio` = 平均量比

**合并策略**：轨道 A 优先，轨道 B 补充
- 今日涨幅、5日/20日涨幅、量比：A 有则用 A，否则用 B
- 上涨占比、涨停数、成分股列表：仅 B 有

### 1.4 核心板块筛选（Step 3）

对所有归属板块计算综合得分：

```
综合得分 = 权重基础分×100 + 趋势加分 + 涨停加分

趋势加分：
  今日涨幅 > 5%  → +25
  今日涨幅 > 3%  → +20
  今日涨幅 > 1%  → +10
  今日涨幅 > 0%  → +5

涨停加分：min(涨停家数 × 3, 15)
```

按得分降序排序，取前 **5 个**核心板块。确保多样性：
- 如果缺少申万行业，从剩余中补一个
- 如果缺少 EM 概念，从剩余中补一个

### 1.5 7 维度评分（Step 4）

对最佳核心板块（排序第 1）进行评分，满分 100 分，通过阈值 **≥55 分**。

| 维度 | 分值范围 | 数据来源 | 评分规则 |
|------|---------|---------|---------|
| 1. 今日涨幅 | 0-25 | 双轨合并 | >5%→25, >3%→20, >1%→15, >0%→10, 负值→max(0, 5+涨幅) |
| 2. 5日趋势 | 0-20 | NeoData 优先，B回填 | >10%→20, >5%→15, >2%→10, >0%→5, 负值→max(0, 5+涨幅) |
| 3. 20日趋势 | 0-10 | NeoData 优先，B回填 | >15%→10, >5%→7, >0%→4, 负值→max(0, 3+涨幅/5) |
| 4. 情绪指标 | 0-15 | 成分股计算 | 上涨比/100×10 + 涨停数×1.5，上限15；无上涨比时用 `涨幅×2` 估算 |
| 5. 量能指标 | 0-10 | 双轨合并 | 量比>2→10, >1.5→8, >1→6, >0.8→4, 否则→2 |
| 6. 相对强度 | 0-10 | 所有核心板块 | 最佳板块涨幅 vs 所有核心板块平均；`max(0, min(5+差值×2, 10))` |
| 7. 板块多样性 | 0-10 | 所有核心板块 | ≥3个板块且≥2个上涨→10；≥1个上涨→5；否则0 |

**最终得分** = 7 个维度之和，权重 **8%**（即 `总分 × 0.08` 计入综合评分）。

### 1.6 reason 输出格式

```
核心板块: 电子, 消费电子, 消费电子零部件及组装 | 今日涨幅+1.69%(15分) | 5日+2.54%(10分) | 20日+19.92%(10分) | 情绪上涨比73%(7.3分) | 量能量比1.16(6分) | 相对强度(8.1分) | 多样性(10分) | 总分66.5分✅
```

## 二、[10] 龙头活跃（V3）

### 2.1 整体流程

```
核心板块列表
  → 对每个板块获取动态龙头（Top8）
  → 计算目标股在板块内的定位（角色/排名/相对平均）
  → 找第一个目标股不是"后排"的板块作为 best_board
  → 计算板块活跃度 + 个股地位 → 总分
  → 输出 CheckResult + 龙头详情列表
```

### 2.2 动态龙头选取（Top 8）

对每个核心板块，取该板块的成分股列表，按以下公式计算综合得分：

```
综合得分 = 近5日涨幅 × 0.6 + 今日涨幅 × 0.4
```

按得分从高到低排序，取前 **8 只**作为该板块的动态龙头。

> **设计意图**：兼顾短期爆发力（今日涨幅 40%）和持续性（5日涨幅 60%），防止一日游的假象龙头。

#### 角色标注（在 Top8 内部细分）

| 角色 | 判定条件 |
|------|---------|
| **情绪龙头** | 综合得分第 1 名 |
| **中军** | 在 Top8 的前半段（前 4 名）中，**市值最大**的那只（如果已是情绪龙头则跳过） |
| **补涨候选** | 今日涨幅 < 2% 但**成交额 > 5000万**（涨幅滞后但放量，有补涨潜力） |
| **活跃股** | 其余进入 Top8 的股票 |

### 2.3 目标股在板块内的定位

目标股的角色由 `_calc_stock_position` 判定，**基于它在全部成分股按今日涨幅排序后的位置**，而非只看是否在 Top8 内。

#### 排名计算

1. 将板块全部成分股按**今日涨幅**从高到低排序
2. 找到目标股在排序后的位置（`rank`）
3. 如果目标股不在成分股列表中（被 MAX_CONSTITUENTS=15 截断），单独获取其数据并 append 到列表，重新排序
4. 计算板块平均涨幅 `avg_chg`
5. 计算相对表现 `vs_avg = 目标股今日涨幅 - avg_chg`

#### 角色判定规则

| 角色 | 判定条件 |
|------|---------|
| **龙头** | 涨幅排名 ≤ 3 **且** 跑赢板块平均 > 2% |
| **中军** | 排名前 40% **且** 跑赢板块平均 ≥ 0 |
| **跟风** | 排名前 70% **且** 跑输板块平均 < 2%；或正涨幅 |
| **滞涨** | 跑输板块平均 > 2%，或排名后 20%，或负涨幅且排名后 50% |
| **后排** | 其余（数据缺失时默认） |

> **注意**：`check_sector_leaders_v3` 会遍历核心板块列表，取**第一个**目标股角色不是"后排"的板块作为 `best_board`。如果所有板块都是"后排"，则取第一个有成分股数据的板块。

### 2.4 活跃度评分（0-60 分）

对 `best_board` 的 Top8 龙头计算：

```
avg_5d   = Top8 平均近5日涨幅
avg_today = Top8 平均今日涨幅
limit_up  = Top8 中涨停家数（≥9.9%）

活跃度评分：
  5日维度：
    avg_5d > 5%   → +25
    avg_5d > 2%   → +18
    avg_5d > 0%   → +10
    否则          → max(0, 5 + int(avg_5d))

  今日维度：
    avg_today > 3% → +20
    avg_today > 1% → +14
    avg_today > 0% → +8
    否则           → max(0, 5 + int(avg_today))

  涨停维度：
    min(涨停家数 × 5, 15)

活跃度上限 60 分。
```

### 2.5 个股地位评分（0-40 分）

| 角色 | 得分 |
|------|------|
| 龙头 | 40 |
| 中军 | 30 |
| 跟风 | 20 |
| 滞涨 | 10 |
| 后排 | 0 |

### 2.6 通过条件

```
总分 = 活跃度 + 个股地位得分

若个股是龙头：活跃度 ≥ 20 即通过（龙头放宽）
若个股是中军/跟风：活跃度 ≥ 30 才通过
若个股是滞涨/后排：不通过
```

权重 **15%**（即 `总分 × 0.15` 计入综合评分）。

### 2.7 龙头详情列表

报告中的 `🏢 板块龙头详情` 展示 `best_board` 的 **Top8 龙头**，格式：

```
序号   名称        今日涨跌幅      近5日涨跌幅
------------------------------------------------------
1    胜蓝股份    📈   +12.78%   📈   +20.61%
2    锴威特      📈    +7.53%   📈   +15.01%
...
近5日正增: 8/8  今日盘中正增: 8/8
```

- 正增统计 = Top8 中涨幅 > 0 的家数 / Top8 总数
- 当目标股（如 300843）因综合得分高而进入 Top8 时，也会出现在列表中

## 三、权重调整（V3 vs V2）

| 审核项 | V2 权重 | V3 权重 | 变化原因 |
|--------|--------|--------|---------|
| [9] 板块趋势向上 | 5% | **8%** | 多维度板块分析更精细，提升重要性 |
| [10] 龙头活跃 | 12% | **15%** | 动态龙头 + 个股地位双重判断，提升重要性 |
| [11] 筹码集中度 | 13% | **7%** | 为前两项让出权重 |

## 四、已知问题修复记录

### 4.1 龙头详情中目标股缺失

**现象**：300843 不在 `leader_details` 列表中。

**根因**：`engine.py` 从 `analyze_stock_sectors` 返回的 `leaders_map` 构造 `leader_details`，但 `leaders_map` 中的 Top8 是**第一次** `_fetch_board_leaders` 的结果。而 `check_sector_leaders_v3` 会**再次**调用 `_fetch_board_leaders`，此时 `board['constituents']` 已被 `_calc_stock_position` append 了目标股，排序结果不同。

**修复**：`engine.py` 改为直接从 `check_sector_leaders_v3` 返回结果的 `raw_value['leader_raw_data']` 构造 `leader_details`，确保与 `leader_lines` 完全一致。

### 4.2 龙头角色重复标注

**现象**：`leader_lines` 中出现两只 `[情绪龙头]`。

**根因**：`_fetch_board_leaders` 就地修改 `constituents` 列表中的 `role` 字段，二次调用时旧角色残留。

**修复**：在 `_fetch_board_leaders` 标注角色前，先 `pop` 清除所有已有的 `role`。

### 4.3 统计数字异常（0/5 0/8）

**现象**：`近5日正增: 0/5  今日盘中正增: 0/8`。

**根因**：V3 的 `check_sector_leaders_v3` 返回的 `raw_value` 中未包含 `dim1_pos_5d` 和 `dim2_pos_today` 字段，`build_report.py` 读取时默认值为 0。

**修复**：
1. 在 `check_sector_leaders_v3` 中计算并补充 `dim1_pos_5d` / `dim2_pos_today` / `leader_count` 到 `raw_value`
2. 在 `build_report.py` 中将硬编码分母 `/5` 改为动态 `leader_count`

### 4.4 目标股 name 为空

**现象**：`leader_lines` 中第一条显示 `[情绪龙头] 今日+12.78%` 但没有股票名。

**根因**：`_fetch_constituent_data` 返回的数据中未包含 `stock_name`；`_calc_stock_position` append 目标股时用 `board_data.get('stock_name')` 回填，但板块数据没有 `stock_name`。

**修复**：在 `_fetch_constituent_data` 中从腾讯实时行情 `rt` 读取 `name` 并放入返回数据。

## 五、数据覆盖说明

### 5.1 7 维度数据覆盖

| 维度 | NeoData 可用时 | NeoData 不可用时（当前状态） |
|------|---------------|---------------------------|
| 今日涨幅 | ✅ 指数直接取 | ✅ 轨道B成分股平均 |
| 5日趋势 | ✅ 指数直接取 | ✅ 轨道B成分股平均回填 |
| 20日趋势 | ✅ 指数直接取 | ✅ 轨道B成分股平均回填 |
| 情绪指标 | ❌ 仅B有 | ✅ 成分股上涨占比 + 涨停数 |
| 量能指标 | ✅ 指数量比 | ✅ 轨道B回填量比 |
| 相对强度 | ✅ 基于最佳板块 | ✅ 基于所有核心板块平均 |
| 板块多样性 | ✅ 基于所有核心板块 | ✅ 基于所有核心板块 |

> 当前 NeoData 服务不可用（代理返回空响应），自动降级到轨道B。

### 5.2 限制与降级

- **成分股数量限制**：`MAX_CONSTITUENTS = 15`，超过时只取前15只计算，可能导致目标股被截断（已处理：截断时单独获取目标股数据并补充）
- **腾讯API限制**：非交易日代理受限，实时行情可能不稳定
- **DB 数据缺失**：EM source 关联数据缺失时，回退到 SINA/THS
